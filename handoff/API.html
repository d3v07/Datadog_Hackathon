Authentication

For the hackathon there's one seeded org and a static demo bearer token. Production uses session cookies + scoped API keys; not built today.

# Every request (except /webhooks/* and the Senso public brief) sends:
Authorization: Bearer "demo_token_acme_corp_2026"

# The server resolves token → orgId, attaches to ctx.org, and scopes all queries.
# If missing or invalid:
HTTP/1.1 401 Unauthorized
{ "error": { "code": "unauthenticated", "message": "Missing or invalid bearer token" } }

Error envelope

All errors share the same shape. code is a stable kebab string — clients switch on this, never on the message.

{
  "error": {
    "code": "validation-failed",      // machine-stable
    "message": "vendor.name is required",  // human-friendly
    "details": {                          // optional, schema-specific
      "path": ["vendor", "name"]
    },
    "requestId": "req_01H2KJ4S7P3F"           // echo of X-Request-Id
  }
}
400validation-failedBody or query did not match the zod schema. details contains the path.
401unauthenticatedMissing or invalid bearer token.
403forbiddenToken valid but lacks scope (e.g. trying to hit an admin endpoint).
404not-foundResource id not in scope of the calling org.
409conflictState transition not allowed (e.g. resolving a change that's already resolved).
422unprocessableSchema parsed but semantics fail (e.g. URL not parseable, severity-override unknown).
429rate-limitedPer-org rate limit hit. Honour Retry-After. (Not enforced in hackathon scope.)
500internalUncaught error. Server logs include requestId.
502upstream-failedNimble, Gemini, Senso, or Slack returned an error or timed out. details.provider names which.

Idempotency

Mutating POSTs (anything that creates a row in ClickHouse or fires an external call) accept an Idempotency-Key header. Same key within 24h returns the cached response.

# Recommended: client UUID v4 generated at the click site
POST /v1/bundles
Authorization: Bearer ...
Idempotency-Key: 0190ab12-7c47-7e88-9c5d-d5e0a1b2c3d4
Content-Type: application/json
{ ... }

Pagination

Cursor-based on the lists that grow: /v1/changes and (later) /v1/vendors. Always limit ≤ 100; default 25. Response shape:

{
  "data": [ ... ],
  "page": {
    "cursor": "01H2KJ4S7P3F",  // pass back as ?cursor=...
    "hasMore": true,
    "limit": 25
  }
}

Demo override headers

Two opt-in headers exist so the tweaks panel can rehearse alternative narratives. The server ignores them unless explicitly set.

X-Demo-Force-Severity: P1 | P2 | P3     // overrides policy eval result for the next scheduled tick on the named vendor
X-Demo-Vendor-Override: vnd_notion    // scopes the override to this vendor only

Server-Sent Events
SSE /v1/stream Long-lived event channel. UI subscribes once on app load to drive live updates: new ChangeReports landing, scheduler ticks, Slack delivery confirmations, entitlement changes. Org-scoped (bearer in query)
Connection
GET /v1/stream?token=demo_token_acme_corp_2026

# Content-Type: text/event-stream
# Keepalive: server emits ":heartbeat\n\n" every 15s
# Client should reconnect with Last-Event-ID on disconnect

Why query param, not header: EventSource doesn't support custom headers. The server resolves ?token= exactly like an Authorization bearer.

import { NimbleClient } from "@nimble/sdk";

const nimble = new NimbleClient({ apiKey: process.env.NIMBLE_API_KEY });

export async function fetchUrl(url: string): Promise<NimbleFetchResult> {
  const resp = await nimble.fetch({
    url,
    render: "static",            // no JS rendering today
    timeoutMs: 12000,
    retries: 1,
  });
  if (!resp.ok) throw new UpstreamError("nimble", resp.error);
  return {
    text: clean(resp.text),     // strip nav/footer/scripts
    html: resp.html,
    sourceUrl: resp.finalUrl,
    fetchedAt: new Date().toISOString(),
    hash: sha256(clean(resp.text)),
  };
}

Gemini · structured diff

Live · 1 call per changed URLsrc/providers/gemini.ts

import { GoogleGenAI } from "@google/genai";
import { ChangeReportSchemaJsonSchema } from "./schemas";

const ai = new GoogleGenAI({ apiKey: process.env.GEMINI_API_KEY });

export async function structuredDiff(before, after, vendor) {
  const resp = await ai.models.generateContent({
    model: "gemini-2.5-pro",
    contents: prompt(before, after, vendor),
    config: {
      responseMimeType: "application/json",
      responseSchema: ChangeReportSchemaJsonSchema,
      temperature: 0.1,
    },
  });
  const parsed = ChangeReportSchema.parse(JSON.parse(resp.text));
  // every change must include ≥1 citation, enforced by zod refine
  return parsed;
}

Senso · publish

Live · 1 publish per ChangeReport · 1 per Bundlesrc/providers/senso.ts

export async function publishChangeReport(cr: ChangeReport): Promise<string> {
  const resp = await fetch("https://api.senso.com/v1/publish", {
    method: "POST",
    headers: {
      Authorization: `Bearer ${process.env.SENSO_API_KEY}`,
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      slug: `${cr.vendor.id}-${cr.detectedAt.slice(0,10)}`,
      type: "change-report",
      payload: cr,     // Senso renders the brief from this JSON
      immutable: true,
    }),
  });
  if (!resp.ok) throw new UpstreamError("senso", await resp.text());
  const { publicUrl } = await resp.json();
  return publicUrl;
}

Slack · routed alert

Live · 1 incoming webhooksrc/providers/slack.ts

export async function postAlert(cr: ChangeReport): Promise<string> {
  const blocks = [
    { type: "header", text: { type: "plain_text",
      text: `${cr.severity} · ${cr.vendor.name} · Redline alert` } },
    { type: "section", text: { type: "mrkdwn",
      text: `*${cr.changes[0].summary}*\n_Policy: ${cr.policyFired.name}_` } },
    { type: "context", elements: [
      { type: "mrkdwn", text: `Impact: $${cr.changes[1].dollarImpact?.annualUsd ?? 0}/yr` }
    ] },
    { type: "actions", elements: [
      { type: "button", text: { type: "plain_text", text: "Open in Redline" },
        url: `${BASE_URL}/changes/${cr.id}`, style: "primary" },
      { type: "button", text: { type: "plain_text", text: "View evidence" },
        url: cr.sensoUrl }
    ] }
  ];
  const resp = await fetch(process.env.SLACK_WEBHOOK_URL, {
    method: "POST", headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ blocks }),
  });
  if (!resp.ok) throw new UpstreamError("slack", await resp.text());
  return resp.headers.get("x-slack-msg-id") ?? "";
}
