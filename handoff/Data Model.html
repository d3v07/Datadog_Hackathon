03 · the schema

Data model, tight.

Eleven entities. Five live in ClickHouse (append-only, immutable rows). Six live in process memory (loaded from seed files at boot). Every interface below is the source of truth for both the API and the database — zod parsers derive from these shapes. Copy-paste straight into src/types.ts.

01

Relationship map.

1 org · everything else scoped to it

Org is the root. Everything else carries orgId and queries scope to it. Vendors carry the catalog; Snapshots / ChangeReports / Actions / AgentRuns are the time-series spine.

Org 1 — n User
Users belong to one org. Roles: procurement · legal · security · owner.
Org 1 — n Vendor
Vendors are catalog rows; vendor.ownerId → User.id sets the route target for slack:@vendorOwner.
Org 1 — n Policy
Loaded from seed/policies.yaml; org-scoped at the file level (single org today).
Vendor 1 — n Snapshot
One snapshot per URL per scan. Append-only; the latest two per (vendorId, url) drive the diff.
Vendor 1 — n AgentRun
One row per scheduled tick (always-on cadence). Carries 7 stage events in the agent_traces table.
AgentRun 1 — 0..1 ChangeReport
Runs that detect a diff produce a ChangeReport. Unchanged runs still record the run with status:"unchanged" so the "last scan" timestamp keeps advancing.
ChangeReport 1 — n Change
Each Change is one categorised diff (data / pricing / sub-processor / terms / SLA / security). Embedded in the report JSON.
Change 1 — n Citation
Mandatory: every Change has ≥1 Citation with verbatim quote + URL + timestamp. No citation, no claim.
Policy 1 — n ChangeReport
The "first-match-wins" rule that fired. Stored as policyFiredId on the report; other matches stored in policyAlsoMatched[].
ChangeReport 1 — n Action
Routed actions (Slack / Jira / email / calendar / payment). Slack & payment make real external calls; others are typed records only.
Org 1 — n EvidenceBundle
Generated on demand. Snapshots a date range × vendor set → Senso publish.

Policy
YAML rule. Loaded at boot, evaluated in process.
Seed YAML → memory 
export interface Policy {
  id: PolicyId;
  orgId: OrgId;
  name: string;
  description?: string;

  // the YAML predicate block, parsed
  when: Record<string, string>;

  then: {
    severity: Severity;
    route: string[];                // e.g. ["slack:#sec-grc", "jira:SEC"]
  };

  version: number;
  isActive: boolean;
  createdBy: UserId;
  createdAt: Iso8601;

  // raw YAML kept for the read-only UI panel
  dsl: string;
}

// Six predicate forms — runtime evaluator
export type PredicateOp =
  | { kind: "matches";   value: string }          // substring match
  | { kind: "contains";  value: string }          // array contains
  | { kind: "gt";        value: number }          // numeric >
  | { kind: "lt";        value: number }          // numeric <
  | { kind: "within";    value: string }          // duration "90d", "12h"
  | { kind: "notIn";     values: string[] }        // "not_in:[a,b,c]"
  | { kind: "trend";     dir: "up" | "down" }; // computed before/after

// AND-only within a `when` block (no OR/NOT in MVP)
Snapshot
A fetched page. Append-only. Two per (vendor,url) drive the diff.
ClickHouse · snapshots 
export interface Snapshot {
  id: SnapshotId;
  orgId: OrgId;
  vendorId: VendorId;
  runId: RunId;
  url: string;
  fetchedAt: Iso8601;

ChangeReport
The deliverable. One row per detected diff. Embeds Change[] and Citation[].
ClickHouse · change_reports 
export interface ChangeReport {
  id: ChangeReportId;
  orgId: OrgId;
  vendorId: VendorId;
  runId: RunId;
  detectedAt: Iso8601;

  snoozedUntil?: Iso8601;

  resolution?: Resolution;

  policyAlsoMatched: PolicyId[];

    action: "renegotiate" | "escalate" | "accept" | "reject";

State machine: new → acknowledged → in-progress → resolved. snoozed branches off any pre-resolved state and auto-returns to its prior state at snoozedUntil.

Citation
The grounding primitive. Verbatim quote + URL + when it was true.
Embedded in Change.citations[] 
export interface Citation {
  url: string;                         // the source page
  quote: string;                       // verbatim text from the source
  section?: string;                    // e.g. "§7.2 — Data Retention"
  fetchedAt: Iso8601;                  // when the quote was true
  country?: string;                    // optional: for sub-processor citations
}

Trust primitive. Gemini's responseSchema requires every Change to have ≥1 Citation; zod re-validates server-side. If the model fails to produce one, the run halts with SchemaViolation.

ClickHouse DDL.

5 tables append-only

Run these once at boot if the table doesn't exist. Single ClickHouse Cloud instance, US-East. All tables use MergeTree; no replication, no replicas, no row-level mutation needed for the demo.

Migration · 5 tablessrc/db/clickhouse.sql

-- ─── snapshots ───────────────────────────────────────
CREATE TABLE IF NOT EXISTS snapshots (
  id              String,
  org_id          String,
  vendor_id       String,
  run_id          String,
  url             String,
  fetched_at      DateTime64(3, 'UTC'),
  source          LowCardinality(String),      -- "nimble" | "seed"
  text            String CODEC(ZSTD(3)),
  hash            FixedString(64),             -- sha256 hex
  source_url      String
)
ENGINE = MergeTree
ORDER BY (org_id, vendor_id, url, fetched_at)
PARTITION BY toYYYYMM(fetched_at);

-- ─── change_reports ──────────────────────────────────
CREATE TABLE IF NOT EXISTS change_reports (
  id                  String,
  org_id              String,
  vendor_id           String,
  run_id              String,
  detected_at         DateTime64(3, 'UTC'),
  severity            Enum8('P1'=1,'P2'=2,'P3'=3),
  state               LowCardinality(String),
  acknowledged_at     Nullable(DateTime64(3, 'UTC')),
  snoozed_until       Nullable(DateTime64(3, 'UTC')),
  resolved_at         Nullable(DateTime64(3, 'UTC')),
  resolution          LowCardinality(Nullable(String)),
  policy_fired_id     String,
  policy_also_matched Array(String),
  changes             String,                       -- JSON of Change[]
  recommendation      String,                       -- JSON
  senso_url           Nullable(String),
  owner_id            String,
  updated_at          DateTime64(3, 'UTC') DEFAULT now64()
)
ENGINE = ReplacingMergeTree(updated_at)     -- last-write-wins for state changes
ORDER BY (org_id, vendor_id, detected_at, id)
PARTITION BY toYYYYMM(detected_at);

-- ─── actions ─────────────────────────────────────────
CREATE TABLE IF NOT EXISTS actions (
  id                String,
  org_id            String,
  change_report_id  Nullable(String),
  kind              LowCardinality(String),
  target            String,
  payload           String,                       -- JSON
  fired_at          DateTime64(3, 'UTC'),
  status            LowCardinality(String),
  external_id       Nullable(String),
  error             Nullable(String),
  updated_at        DateTime64(3, 'UTC') DEFAULT now64()
)
ENGINE = ReplacingMergeTree(updated_at)
ORDER BY (org_id, change_report_id, fired_at, id)
PARTITION BY toYYYYMM(fired_at);

-- ─── agent_runs ──────────────────────────────────────
CREATE TABLE IF NOT EXISTS agent_runs (
  id                String,
  org_id            String,
  vendor_id         String,
  started_at        DateTime64(3, 'UTC'),
  ended_at          Nullable(DateTime64(3, 'UTC')),
  duration_ms       Nullable(UInt32),
  status            LowCardinality(String),      -- running | unchanged | changed | failed
  change_report_id  Nullable(String),
  trigger           LowCardinality(String),      -- scheduled | admin | first-scan
  error_stage       LowCardinality(Nullable(String)),
  error_code        Nullable(String),
  error_message     Nullable(String)
)
ENGINE = ReplacingMergeTree(started_at)
ORDER BY (org_id, vendor_id, started_at, id)
PARTITION BY toYYYYMM(started_at);

-- ─── agent_traces ────────────────────────────────────
CREATE TABLE IF NOT EXISTS agent_traces (

ENGINE = MergeTree
ORDER BY (run_id, at)
PARTITION BY toYYYYMM(at);

Query patterns what the API actually runs

Portfolio statssrc/db/queries.ts

-- vendorCount
SELECT count() FROM vendors_cache;            -- in-memory; backed by seed/vendors.json

-- changes in window (e.g. last 7d)
SELECT count()
FROM change_reports FINAL
WHERE org_id = {orgId:String}
  AND detected_at >= now() - INTERVAL 7 DAY;

-- open P1
SELECT count()
FROM change_reports FINAL
WHERE org_id = {orgId:String}
  AND severity = 'P1'
  AND state NOT IN ('resolved', 'snoozed');

-- $ at risk (sums dollarImpact across all open P1/P2)
SELECT sum(JSONExtractFloat(changes, '$[*].dollarImpact.annualUsd'))
FROM change_reports FINAL
WHERE org_id = {orgId:String}
  AND severity IN ('P1', 'P2')
  AND state NOT IN ('resolved', 'snoozed');
⚠

Always use FINAL on ReplacingMergeTree reads

State transitions (acknowledge / snooze / resolve) work by inserting a new row with the same primary key and a fresher updated_at. ClickHouse merges these in the background, but reads can see both rows until merge. Use SELECT ... FROM change_reports FINAL WHERE ... to always get the latest. FINAL is slower but fine at our row count (≤10k for the hackathon).

snapshots.json & change-reports.json

Seeded "before" snapshots (so the live Nimble fetch produces a hash mismatch) and 4–5 historical change reports across the background fleet (so the activity feed isn't empty on page load). Same shape as the API responses; load straight into ClickHouse.

Seed loader sketchsrc/seed/loader.ts

export async function loadSeeds() {
  const orgs     = readJson("seed/orgs.json");
  const users    = readJson("seed/users.json");
  const vendors  = readJson("seed/vendors.json");
  const policies = parseYaml(readFile("seed/policies.yaml"));

  // in-memory caches always replace
  cache.set("orgs",     orgs);
  cache.set("users",    users);
  cache.set("vendors",  vendors);
  cache.set("policies", policies);

  // CH tables: idempotent on `source='seed'` for snapshots, ids for change_reports
  const existingSnaps = await ch.query({
    query: "SELECT count() AS n FROM snapshots WHERE source = 'seed'",
    format: "JSONEachRow",
  });
  if (existingSnaps[0].n === 0) {
    await ch.insert({ table: "snapshots",
